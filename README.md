# task-queue

Типизированная локальная очередь задач для Node.js 20+ на чистом TypeScript, без внешних зависимостей.

## Краткое описание

`TaskQueue` принимает асинхронные задачи и выполняет их с ограничением параллельности. Повторные постановки по тому же ключу дедуплицируются, упавшие задачи повторяются с экспоненциальной задержкой и джиттером, а корректное завершение (`shutdown`) дожидается активных задач и отменяет только ожидающие.

## Возможности

- **Лимит параллельности** — одновременно выполняется не более `concurrency` задач.
- **Дедупликация по ключу** — повторный `add` с уже выполняющимся ключом возвращает существующий промис, задача не запускается повторно.
- **Retry с exponential backoff и jitter** — задержка `baseDelayMs * 2 ** (attempt - 1)`, ограничивается `maxDelayMs`, размывается множителем в диапазоне `[1 - jitterRatio, 1 + jitterRatio]`.
- **Graceful shutdown** — отменяет ожидающие задачи (`TaskCanceledError`), дожидается активных, после закрытия новые `add` отклоняются `QueueClosedError`.
- **Статистика** — счётчики `enqueued`, `started`, `deduplicated`, `retried`, `failed`, `canceled`.

## Пример использования

```ts
import { TaskQueue } from "./src/index.js";

const queue = new TaskQueue({
  concurrency: 2,
  retries: 3
});

const result = await queue.add("user:42", async ({ signal, attempt }) => {
  return "ok";
});

await queue.shutdown();
```

## Опции `TaskQueueOptions`

| Опция | По умолчанию | Описание |
|---|---|---|
| `concurrency` | — (обязательна) | Максимум одновременных задач, целое > 0. |
| `retries` | `0` | Число дополнительных попыток после первой. |
| `baseDelayMs` | `100` | Базовая задержка перед повтором. |
| `maxDelayMs` | `30_000` | Верхняя граница задержки. |
| `jitterRatio` | `0.2` | Доля случайного размытия задержки (≥ 0). |
| `clock` | — | Хук для тестов: подмена `sleep` и `random`. |

## Архитектура

- **Очередь хранится в памяти** — обычный массив `QueueItem`, без внешних брокеров и хранилищ.
- **Дедупликация через `Map`** — `inFlight: Map<TaskKey, QueueItem>`; ключ присутствует всё время жизни задачи (ожидание, выполнение, ретраи) и удаляется только после успеха, финальной ошибки или отмены.
- **Лимит параллельности через `activeCount`** — диспетчер `drain()` запускает задачи, пока `activeCount < concurrency` и очередь непуста.
- **Ретраи выполняются внутри активного слота** — повторные попытки идут в рамках уже занятого слота и не отнимают дополнительную параллельность.
- **Shutdown отменяет только ожидающие задачи** — активные не прерываются и доводятся до конца; промис `shutdown()` резолвится, когда не осталось активных задач. Ошибка активной задачи не приводит к отклонению `shutdown()`.

## Почему без Nest.js, Redis и Kafka

Nest.js, Redis и Kafka **намеренно не используются**. Задача — показать чистую реализацию базовой логики очереди (параллельность, дедупликация, ретраи, graceful shutdown) на стандартных средствах Node.js (`Promise`, таймеры, `AbortController`/`AbortSignal`), без внешних библиотек для очередей, локов и ретраев.
